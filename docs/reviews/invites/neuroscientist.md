## 评审背景

EvoAgent 的记忆系统设计了遗忘曲线机制：

```
Session 价值评估 = 唯一性 × 引用计数 × 成功率

高价值 Session → keepForever 永不过期
低价值 Session → 定期清理
```

技能系统也有类似机制：

```
技能使用统计:
- timesUsed: 15
- timesSucceeded: 14
- timesFailed: 1
- avgDuration: 45000
```

## 核心问题

### 1. 技能学习曲线

**问题**：
- 技能是否应该有"学习曲线"？
- 新技能是否需要"试用期"？
- 技能如何从 "draft" 状态过渡到 "validated"？

### 2. 技能"固化"机制

类似记忆巩固：

```
Session → (访问增强) → 巩固为 Knowledge
Agent使用技能 → (成功使用) → 技能固化
```

**问题**：
- 我们的 "validated" 状态是否等同于技能的"巩固"？
- 是否需要根据使用频率动态调整技能评分？
- 失败次数过多是否自动"降级"技能？

### 3. 技能"遗忘"机制

类似 Session 的价值评估：

```
技能价值 = 使用频率 × 成功率 × 最近使用时间
```

**问题**：
- 技能是否应该有"遗忘"机制？
- 30天未使用的技能是否应该被标记？
- 废弃技能如何选择是"完全删除"还是"归档保留"？

### 4. 技能与知识的学习协同

```
用户反馈: "这个 React 组件创建技能很好用"
    ↓
1. 更新技能评分
2. 提取经验到 Knowledge
3. 优化技能模板
```

**问题**：
- 这个反馈循环是否合理？
- 如何确保技能和知识保持同步？
- 如何避免过度优化（overfitting）？

## 心理学视角的技能设计

### 技能命名

设计：使用清晰、描述性的名称

```
react-component-creation  ← 清晰
api-error-handling        ← 清晰
login-form-creation      ← 清晰
```

**问题**：
- 命名规范是否符合认知科学？
- emoji 图标是否有帮助？
- 是否需要更短的别名？

### 技能组合

设计：技能可以组合使用

```
技能A: 创建React组件
技能B: 添加样式
技能C: 添加表单验证

→ 可以组合使用: A + B + C 创建完整表单组件
```

**问题**：
- 技能组合的粒度如何设计？
- 如何避免组合爆炸？
- 如何处理技能间的冲突？
