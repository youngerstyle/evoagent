# 技术圆桌会议技能 - 快速指南

## 快速使用

### 场景1: 组织新功能设计评审

```bash
# 1. 查看技能定义
cat skills/manual/technical-roundtable/SKILL.md

# 2. 准备评审材料
# 更新 docs/reviews/{模块名}-{日期}.md

# 3. 使用邀请模板
cat skills/manual/technical-roundtable/templates/invite.md
```

### 场景2: 邀请特定专家

根据设计文档中的专家列表：

| 评审内容 | 推荐专家 |
|---------|---------|
| 技能系统 | ClawdBot作者（技能系统专家） |
| 记忆系统 | OpenMemory、PageIndex、神经科学家 |
| 架构/并发 | Mauns开发、SRE专家 |
| 数据/存储 | 数据库专家 |

### 场景3: 记录评审结果

```markdown
# {模块名} 专家评审会 - Round {N}

**日期**: 2025-01-30
**版本**: v{version}
**参会专家**: ...

## 专家反馈

### 👤 专家1
**P0**: ...
**P1**: ...

## 评审结果
- [ ] 通过
- [ ] 需要整改

## 整改计划
| 问题 | 负责人 | 截止时间 |
|------|--------|----------|
```
